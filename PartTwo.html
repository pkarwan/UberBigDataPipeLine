Part Two: Exploration of northwind.sqlite Database¶

In this document we'll be exploring the performance of the nine employees listed in the northwind.sqlite database. The 'northwind' database is sample database created by Microsoft for let's say, abc company having business model related to food and beverages distribution. This database contains details for the products, categories, customers, orders, inventory, purchasing, suppliers, shipping and also employees!

For this Project, we'll explore more on employees based on below criteria such as:

  • how much quantity they sold
  • products of total amount sold
  • how many products sold
  • employee's regional performance
  • employee's overall performance over the time,
  • employee's strengths, and weaknesses.

To get started, first we need to import below libraries.

  1. import sqlite3 - This module allows accessing the sqlite3 database using a nonstandard variant of the SQL query language.
  2. import pandas as pd - pandas is an open source data analysis library built on top of the Python programming language.
  3. import numpy as np - NumPy is the fundamental package for scientific computing in Python.
  4. from collections import Counter - Collection module implements specialized container datatypes and Counter dict subclass for counting hashable objects
  5. import matplotlib.pyplot as plt - This is a collection of functions that make matplotlib work like MATLAB. Each pyplot function makes some change to a figure
  6. import seaborn as sns - Seaborn is a library for making statistical graphics in Python. It builds on top of matplotlib and integrates closely with pandas data structures.

Now, let's deep dive into database tables and take a look at the data provided to get a meaningful insights from a business model.

from collections import Counter

def most_common_item(List):
    Occurence_count = Counter(List) #counts occurrence of each sold item
    return Occurence_count.most_common(1)[0][0] #returns most common

Now we need to create a couple of new variables to find the exact information we're looking for. First, we'll use the OrderDate variable and extract only the first four characters of the string to create an OrderYear variable. Then we'll define a function that uses UnitPrice, Quantity, and Discount to calculate the sale revenue (including discount) for each order. We'll call this variable OrderRevenue. Then we'll calculate the without discount revenue and call this TotalRevenue.

In [73]:
sales['OrderYear'] = sales.OrderDate.str[:4] #only read first four chars of OrderDate variable to make year

def tot_rev(unitprice, quantity, discount): #calculate discounted total
    subtot = unitprice * quantity
    disc = subtot * discount
    tot = subtot - disc
    return tot

sales['OrderRevenue'] = tot_rev(sales.UnitPrice, sales.Quantity, sales.Discount) #apply discounted total func
sales['TotalRevenue'] = sales.UnitPrice * sales.Quantity #caluculate non-discounted total
